## 一、插件配置管理

### 1.1 插件ID命名规范

**问题案例**：止观插件ID不匹配警告
- **现象**：`plugin id mismatch (manifest uses "zhiguan", entry hints "openclaw-plugin")`
- **原因**：`openclaw.plugin.json` 中定义的ID与 `openclaw.json` 中 `plugins.entries` 的键名不一致
- **解决方案**：
  ```json
  // openclaw.plugin.json
  { "id": "zhiguan-ai" }
  
  // openclaw.json - plugins.entries
  { "zhiguan-ai": { "enabled": true } }
  ```

**规范要求**：
1. 插件ID必须全局唯一，建议使用反向域名风格（如 `com.company.plugin-name`）
2. `openclaw.plugin.json` 中的 `id` 必须与 `openclaw.json` 中 `plugins.entries` 的键名完全一致
3. 避免使用过于通用的ID（如 `plugin`、`app` 等）

### 1.2 插件重复ID问题

**问题案例**：飞书插件重复ID警告
- **现象**：`feishu: duplicate plugin id detected`
- **原因分析**：这是正常配置结构，非真正重复
  - `channels.feishu` - 频道连接配置
  - `plugins.entries.feishu` - 插件启用配置
  - `plugins.installs.feishu` - 插件安装信息
- **处理方式**：此警告可忽略，或联系OpenClaw开发团队优化检测逻辑

**配置结构说明**：
```json
{
  "channels": {
    "feishu": { /* 频道配置 */ }
  },
  "plugins": {
    "entries": {
      "feishu": { "enabled": true }  // 插件启用
    },
    "installs": {
      "feishu": { /* 安装信息 */ }   // 安装记录
    }
  }
}
```

### 1.3 插件加载路径配置

**配置位置**：`plugins.load.paths`
```json
{
  "plugins": {
    "load": {
      "paths": [
        "/root/.openclaw/extensions/kimi-claw",
        "/root/.openclaw/workspace/zhiguan-plugin"
      ]
    }
  }
}
```

**注意事项**：
1. 路径必须是绝对路径
2. 每个路径必须包含有效的 `openclaw.plugin.json` 文件
3. 插件目录变更后需要重启OpenClaw生效

## 二、健康检查与监控

### 2.1 健康检查脚本

**脚本位置**：`/root/.openclaw/workspace/monitor-openclaw.sh`

**功能说明**：
- 检查OpenClaw Gateway运行状态
- 自动重启异常服务
- 日志文件轮转管理
- 磁盘空间监控

**Cron配置**：
```bash
# 每2小时执行一次（可根据需要调整）
0 */2 * * * /root/.openclaw/workspace/monitor-openclaw.sh
```

**检查频率调整方法**：
```bash
# 查看当前cron任务
openclaw cron list

# 更新执行频率（每2小时 = 7200000毫秒）
openclaw cron update <job-id> --schedule '{"kind":"every","everyMs":7200000}'
```

### 2.2 监控指标

| 指标 | 正常状态 | 异常处理 |
|------|---------|---------|
| Gateway状态 | running | 自动重启 |
| 日志大小 | < 100MB | 自动轮转 |
| 磁盘使用率 | < 80% | 发送警告 |

## 三、配置文件管理

### 3.1 主配置文件

**文件位置**：`/root/.openclaw/openclaw.json`

**备份策略**：
- 系统自动创建 `.bak` 备份文件
- 重要修改前手动备份：
  ```bash
  cp /root/.openclaw/openclaw.json /root/.openclaw/openclaw.json.bak.$(date +%Y%m%d%H%M%S)
  ```

**配置验证**：
```bash
# 检查JSON格式
openclaw config validate

# 查看配置警告
openclaw skills list
```

### 3.2 敏感信息保护

**包含敏感信息的配置项**：
- `env.KIMI_API_KEY`
- `channels.feishu.appSecret`
- `gateway.auth.token`
- `plugins.entries.*.config.*.token`

**安全措施**：
1. 配置文件权限设置为 `600`（仅所有者可读写）
2. 定期轮换API密钥和Token
3. 不要将配置文件提交到Git仓库
4. 使用环境变量覆盖敏感配置（如支持）

## 五、常见问题处理

### 5.1 插件配置问题

**症状**：启动时出现插件警告
**处理流程**：
1. 查看详细警告信息：`openclaw skills list`
2. 检查插件ID一致性
3. 验证插件目录结构
4. 重启OpenClaw生效

### 5.2 文件传输问题

**症状**：文件发送失败或用户未收到
**处理流程**：
1. 确认文件存在：`ls -la <file-path>`
2. 使用正确的发送方式：
   - 飞书：使用 `message` 工具的 `filePath` 参数
   - 其他渠道：使用对应渠道的文件传输功能
3. 避免使用 `kimi_upload_file` 发送工作文件

### 5.3 音效播放问题

**症状**：网页版Demo无声音
**原因**：浏览器安全策略限制本地音频文件加载
**解决方案**：
1. 使用Base64编码嵌入音频
2. 部署到HTTPS服务器
3. 启动本地HTTP服务器访问

## 六、安全最佳实践

### 6.1 定期维护任务

| 任务 | 频率 | 命令/操作 |
|------|------|----------|
| 检查配置警告 | 每周 | `openclaw skills list` |
| 查看日志文件 | 每周 | `ls -lh /tmp/openclaw/` |
| 更新Skills | 每月 | `npx clawhub update` |
| 轮换API密钥 | 每季度 | 手动更新配置 |
| 备份配置文件 | 每月 | 复制到安全位置 |

### 6.2 应急响应

**Gateway无法启动**：
1. 检查配置文件JSON格式
2. 查看日志：`cat /tmp/openclaw/openclaw-*.log`
3. 恢复备份配置
4. 手动重启：`openclaw gateway restart`

**插件加载失败**：
1. 检查插件目录权限
2. 验证 `openclaw.plugin.json` 格式
3. 重新安装插件
