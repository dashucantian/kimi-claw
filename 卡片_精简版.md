# 卡 搭-X：用AI助手协作开发微信小程序（精简版）

## 设计原则
- 八格表单结构（触发→产出→输入→提示词→步骤→验收→排查→升级）
- 明暗双线设计（明线卖省时间，暗线藏方法论）

---

## 场景故事

2026年2月，产品经理小林想做一个正念冥想小程序。他懂产品逻辑，但完整的小程序开发对他来说是个黑箱。

以前他可能会花两周啃文档、踩坑。但这次他打开飞书，对着止观AI机器人说："我想做一个四阶段禅修的小程序。"

3天后，止观小程序v1.0上线了。

这不是"AI代替开发者"，而是"开发者有了个24小时在线的技术搭档"。

## 八格表单

### 1. 什么时候用？

- 有产品想法，但技术实现路径不清晰
- 需要快速验证MVP，不想从零学框架
- 卡在具体技术点（音频播放、本地存储）
- 想学习开发，但需要实时指导

### 2. 做出来是什么？

**产出**：可运行的微信小程序代码 + 项目文档

**给谁**：产品经理/创业者/初级开发者

### 3. 需要准备什么？

| 类型 | 内容 | 说明 |
|------|------|------|
| 必须 | 产品需求描述 | 功能列表、页面流程 |
| 必须 | 微信开发者工具 | 官方免费下载 |
| 必须 | AI助手对话环境 | 如飞书机器人 |
| 建议 | 参考竞品截图 | 降低沟通成本 |
| 建议 | 音频素材 | 或让AI生成TTS |

**关键原则**：需求描述越具体，AI输出越精准。

### 4. 怎么跟AI说？

```
## 背景
我想开发一个微信小程序，定位是【正念冥想工具】。
目标用户是【想要开始冥想的职场人】。

## 功能需求
1. 【首页】：选择练习时长和模式
2. 【练习页】：倒计时+四阶段引导+音效
3. 【完成页】：评分+备注
4. 【统计页】：历史记录

## 技术要求
- 微信小程序原生框架
- Web Audio API
- 微信本地存储

## 输出要求
1. 完整页面代码
2. 结构说明
3. 关键实现解释
4. 优化建议

## 约束条件
- 代码可直接运行
- 标注外部资源需求
- 说明API限制
```

### 5. 具体怎么操作？

| 步骤 | 动作 | 耗时 |
|------|------|------|
| 1 | 描述产品需求 | 10分钟 |
| 2 | AI生成初版代码 | 5分钟 |
| 3 | 导入开发者工具 | 3分钟 |
| 4 | 测试并记录问题 | 10分钟 |
| 5 | 向AI反馈问题 | 5分钟 |
| 6 | AI修复并迭代 | 5分钟 |
| 7 | 验证修复结果 | 5分钟 |
| 8 | 重复直到满意 | 3-5轮 |

**总耗时**：首次2-3小时，后续每轮10-15分钟

### 6. 怎么算做完？

- [ ] 首页可选择时长和模式
- [ ] 练习页倒计时准确
- [ ] 音频正常播放
- [ ] 完成页可提交评分
- [ ] 统计页显示历史
- [ ] 开发者工具无报错

**修正指令**：
- "倒计时后台暂停了" → 添加后台音频播放
- "底部导航被顶起" → 使用fixed定位
- "音频无法循环" → 检查onEnded事件

### 7. 出错了怎么办？

| 问题 | 症状 | 解法 |
|------|------|------|
| 代码跑不通 | 控制台报错 | 复制报错给AI；检查API版本 |
| 样式差距大 | 布局错乱 | 提供截图；明确布局方式 |
| 音频无法播放 | 没声音 | 检查路径；确认权限；换Web Audio |
| 数据没保存 | 刷新丢失 | 检查storage调用；确认键名 |
| 真机表现不同 | 模拟器正常 | 优先真机；检查版本差异 |

### 8. 以后能自动吗？

| 阶段 | 方式 | 你的角色 |
|------|------|----------|
| 手动 | 每次对话描述 | 完整描述+验证 |
| 半自动 | 建立知识库 | 描述变更点 |
| 自动 | 接入低代码平台 | 配置参数+验收 |

## 高手心法

**真正的效率提升，来自"对话式开发"的飞轮效应**

传统开发是线性的：需求→设计→开发→测试→交付

AI协作开发是循环的：
```
描述 → 生成 → 验证 → 反馈 → 修复 → 循环
```

每一次循环，你都在：
1. **校准理解**：AI越来越懂你的需求
2. **沉淀知识**：问题与解决方案成为文档
3. **提升表达**：描述需求的能力越来越强

这就是「止观开发飞轮」：
**觉察 → 行动 → 调整 → 反思**

**记住公式**：
```
开发效率 = 需求清晰度 × AI理解准确度 × 验证反馈速度
```

当你觉得AI"不够聪明"时，先问自己：
1. 功能流程描述够清晰吗？
2. 提供了足够上下文吗？
3. 反馈的问题够具体吗？

你不是在"让AI帮你写代码"，你是在训练一个懂你的技术搭档。

## 附：止观开发飞轮实战数据

- 单次迭代：30分钟 → 5分钟（6倍提升）
- 沟通轮次：5轮 → 1轮（5倍提升）
- 返工率：40% → 5%（8倍提升）

**关键经验**：
1. 音量感知非线性，需指数级调整
2. 文件上传需要确认机制
3. 消息队列需要优先级管理
4. 文档和开发应并行进行
