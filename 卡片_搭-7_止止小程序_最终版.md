# 卡 搭-7：用AI助手开发「止止」小程序——数字时代的心智锚点

## 场景故事

每天，地铁摇晃中刷短视频，电脑屏幕前切换文档与消息，睡前躺在床上无意识滑动手机——时间在信息流中无声流逝，到站时想不起看了什么，下班时觉得空虚，入睡时承诺"再看最后一个"重复二十遍。

宗国也曾如此。藏地修行八载、岩洞闭关三年的他，深知现代人困境：不是不想修行，而是抽不出三个月去深山；不是不想专注，而是被数字洪流裹挟难以自拔。

2026年2月，他打开飞书，对止观AI机器人说："我想做一个【止止】小程序，让人们在碎片时间练习身心收放，像串珍珠一样把散落的片刻连起来。"

3天后，【止止】v1.0上线。不是让人逃离数字生活，而是教他们在碎片中练习收放自如——等电梯的三十秒调息，会议间隙的五分钟止念，睡前的十分钟观心。智能记录串成珍珠，看见自己的模式；每一次练习，都是心智操作系统的迭代更新。

这不是"AI代替修行者"，而是"修行者有了个24小时在线的智能伙伴"。

### 6. 怎么算做完？（验收标准）

- [ ] 首页可选择3/5/10分钟和对应场景
- [ ] 四阶段流程完整，切换流畅
- [ ] 磬声/引磬等音效正常播放
- [ ] 后台切换时音频不中断
- [ ] 完成页可评分、备注、保存记录
- [ ] 统计页显示连续天数和成长曲线
- [ ] 微信开发者工具无报错
- [ ] 真机预览基本可用

**修正指令示例**：
- "音频在后台暂停了" → "请配置后台音频播放权限，确保切换应用时音频继续"
- "阶段切换没有提示" → "请在每个阶段开始和结束时添加语音引导"
- "数据没有保存" → "请检查localStorage写入逻辑，添加存储成功回调"

### 7. 出错了怎么办？（失败排查）

| 问题 | 症状 | 解法 |
|------|------|------|
| AI不理解"止观四阶段" | 生成的流程不符合禅修逻辑 | 提供详细引导语文本；解释每阶段目的 |
| 音频无法循环播放 | 播放一次后停止 | 检查onEnded事件；配置循环播放逻辑 |
| 后台音频中断 | 切换应用后静音 | 配置微信小程序后台播放权限 |
| 数据记录丢失 | 关闭小程序后记录消失 | 检查localStorage键名；添加读取验证 |
| 真机与模拟器表现不同 | 模拟器正常，真机异常 | 优先以真机为准；检查微信版本差异 |

**特殊场景：修行逻辑与代码实现的冲突**
- 问题：AI生成的代码过于"效率导向"，缺少"觉察引导"
- 解法：明确告知"这不是工具，是修行伴侣"；要求在每个交互中加入"暂停、感受、觉察"的提示

## 高手心法

**【止止】的本质不是小程序，是"心智开发飞轮"的方法论**

传统的修行是线性的：拜师→闭关→证悟→下山。每个环节之间是"多年苦修"的距离。

【止止】用AI技术重构了修行流程，变成循环的飞轮：

```
觉察状态 → 行动练习 → 调整方法 → 反思沉淀 → 再次觉察
     ↑                                              ↓
     └──────────── 心智操作系统迭代 ────────────────┘
```

每一次循环，你都在做三件事：
1. **觉察**：在数字洪流中看见自己的散乱
2. **行动**：用碎片时间练习收放（等电梯、会议间隙、睡前）
3. **沉淀**：智能记录串成珍珠，看见模式与成长

这就是「止观开发飞轮」在修行场景的应用：
- **觉察**：快速看见当下的身心状态
- **行动**：立即启动对应的练习方案
- **调整**：根据反馈优化引导策略
- **反思**：记录感悟，形成个人修行档案

**记住一个公式**：

```
心智成长 = 觉察频率 × 练习深度 × 反思质量
```

AI的智能化记录和提醒已经很强了。差距在于：你是否愿意在碎片中停下来，是否诚实地面对自己的散乱，是否持续地练习收放。

当你觉得"没时间修行"时，先问自己三个问题：
1. 等电梯的30秒，我是否可以调息而不是刷手机？
2. 会议间隙的5分钟，我是否可以止念而不是回消息？
3. 睡前的10分钟，我是否可以观心而不是看视频？

【止止】不是让你逃离数字生活，而是教你在数字生活中练习收放自如。手机还在，电脑还在，但你不再是它们的奴隶。

你不是在"用一个小程序"，你是在训练一个懂你的智能修行伙伴。
