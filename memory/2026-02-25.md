## 错误描述

创建飞书文档成功，但多次尝试写入内容均返回 HTTP 400 错误，导致文档只有标题没有内容。

---

## 根本原因分析

1. **未验证写入结果**：创建文档后没有立即读取验证内容是否成功写入
2. **重复尝试相同方法**：连续使用 `write` 和 `append` 操作，都失败了但没有换策略
3. **没有备选方案**：当 API 写入失败时，没有立即提供替代方案（如直接上传文件）
4. ** SKILL.md 限制未注意**：飞书文档的 `write` 操作不支持 Markdown 表格，但我在内容中使用了表格

---

## 正确的操作流程（改进后）

### 飞书文档操作标准流程

```
1. 创建文档 → 获取 doc_token
   ↓
2. 尝试写入内容（简化格式，避免表格）
   ↓
3. 立即读取验证内容是否成功写入
   ↓
4. 如果写入失败：
   a. 尝试分段 append 写入
   b. 如果仍失败，立即生成本地文件
   c. 通过 kimi_upload_file 上传给用户
   ↓
5. 告知用户文档链接 + 备选文件
```

---

## 关键注意点

### 飞书文档 feishu_doc 工具限制

| 操作 | 限制 | 解决方案 |
|------|------|----------|
| write | 不支持 Markdown 表格 | 使用列表代替表格，或创建后手动添加表格 |
| write | 长内容可能失败 | 分段使用 append |
| append | 每次只能追加，不能修改 | 确保内容顺序正确 |

### 验证步骤（必须执行）

写入内容后**必须**执行：
```json
{ "action": "read", "doc_token": "XXX" }
```

确认 `block_count` > 1 且内容不为空。
